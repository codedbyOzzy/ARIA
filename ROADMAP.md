# ARIA Roadmap

The development of ARIA is structured around rapid, feature-dense sprints. The goal is to aggressively push the boundaries of what a personal desktop AI can achieve while maintaining absolute local privacy.

## Phase 1: The Core Foundation (Completed)
- Transition to the ARIA Agentic Router Engine.
- Sub-second latency optimization for basic tool calls.
- Base Desktop Control (App launching, window management) and Smart FAB overlay.

## Phase 2: Autonomous Intelligence
- **Night Shift Engine:** Deploying fully autonomous agents that run for hours to complete complex research tasks while the user sleeps.
- **Deep App Integrations:** Direct hooks into Discord, Spotify, and YouTube Music to control and read state without generic UI scraping.
- **Human-in-the-Loop Protocol:** Ensuring destructive commands always require explicit UI approval.

## Phase 3: The Swarm Expansion
- **Multi-Agent Orchestration:** Spin up isolated agents for parallel processing (e.g., a "Coder Agent" debugging while a "Researcher Agent" gathers docs).
- **Advanced Screen Vision:** Seamlessly processing visual data contextually without taking screenshots to the cloud.
- **Cross-App Overlay Enhancements:** Ubiquitous accessibility from any game, browser, or editor without breaking user flow.

## Phase 4: Extreme Autonomy
- **Self-Healing Routines:** ARIA detects errors in its own background scripts and attempts to resolve them.
- **Predictive Pre-fetching:** Loading context before the user even types, based on current screen activity and historical patterns.

*Note: This roadmap is subject to rapid change based on real-world usage and the evolution of underlying local LLM capabilities (like Ollama and Llama 3).*
