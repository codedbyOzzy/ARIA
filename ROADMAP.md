# ARIA Roadmap

The development of ARIA is structured around rapid, feature-dense sprints. The goal is to aggressively push the boundaries of what a personal desktop AI can achieve.

## Phase 1: The Core Foundation (Completed)
- Transition from FRIDAY's event-bus to the ARIA Tri-Core Engine.
- Sub-second latency optimization (< 1.2s first token).
- Base Desktop Control (App launching, volume, window management).

## Phase 2: Ambient Awareness
- **Clipboard Intelligence:** Context-aware chips when `Alt+Space` is triggered with data in the clipboard.
- **Background Tasks:** Ability to schedule long-running background tasks.
- **Daily Briefing Engine:** Aggregating data for proactive morning summaries.

## Phase 3: Deep Desktop Integration
- **Advanced File Vision:** Seamlessly dragging and dropping files into the ARIA panel for instant contextual analysis.
- **UI Automation:** Interacting directly with specific application interfaces via accessibility APIs.
- **Extended Tool Suite:** Deep integration with Steam, Telegram, and complex browser workflows.

## Phase 4: Extreme Autonomy
- **Self-Healing Routines:** ARIA detects errors in its own background scripts and attempts to resolve them.
- **Predictive Pre-fetching:** Loading context before the user even types, based on current screen activity and historical patterns.

*Note: This roadmap is subject to rapid change based on real-world usage and the evolution of underlying LLM capabilities.*
